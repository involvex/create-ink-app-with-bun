# Plan: Enhanced Bun + Ink TypeScript Template

## Problem Analysis

The parsing errors (`; expected`) are caused by:

1. **Import extensions**: Files use `.js` extensions in TSX files (e.g., `import ... from './app.js'`)
2. **ESLint config file**: Using `.ts` extension can cause parsing issues with ESLint's flat config system
3. **Missing description placeholder**: `%DESCRIPTION%` not handled in copyWithTemplate

## Implementation Plan

### Phase 1: Fix Parsing Errors (Critical - Resolves ESLint Errors)

1. **Update all imports in TSX files** - Remove `.js` extensions from local imports:
   - `templates/bun/src/app.tsx`: `import SelectInput from './components/select-input.js'` → `./components/select-input`
   - `templates/bun/src/cli.tsx`: All `.js` imports → no extension
   - `templates/bun/src/commands/*.tsx`: `.js` imports → no extension

2. **Rename ESLint config** - Change `eslint.config.ts` → `eslint.config.mjs`

### Phase 2: ESLint Configuration

1. Create `templates/bun/eslint.config.mjs` with:
   - TypeScript ESLint plugin (typescript-eslint)
   - React and React Hooks rules (eslint-plugin-react, eslint-plugin-react-hooks)
   - Strict rules: no-unused-vars, no-explicit-any, no-console
   - File patterns for `.ts` and `.tsx` only (no JavaScript files)
   - React Hooks exhaustive deps rule
