# Plan: Simplify create-ink-app-with-bun to Bun-only default

## Summary

Remove JS and TS templates, make Bun the default and only template. Add bun-types, improve bunfig.toml for Ink, and remove @types/node from template.

---

## Tasks

### 1. Modify cli.js

- Remove `--typescript` flag
- Remove `--bun` flag (no longer needed)
- Remove `--template` flag (no longer needed)
- Update help text to reflect Bun-only
- Remove typescript from prompts (not relevant)
- Update success message to show `bun run` commands instead of `npm run`

### 2. Modify index.js

- Simplify `createInkApp` function - remove `typescript`, `bun`, `template` parameters
- Remove logic that selects between templates (always use `templates/bun`)
- Remove JS and TS template copy tasks, keep only Bun tasks
- Update output messages to show Bun commands

### 3. Update templates/bun/\_package.json

- Remove `@types/node` from devDependencies
- Add `bun-types` to devDependencies (or `@types/bun` if bun-types not preferred)
- Keep `@types/react`, `@types/ink`, etc. as they are still needed for React types

### 4. Update templates/bun/tsconfig.json

- Replace `"types": ["node"]` with `"types": ["bun-types"]` (or `@types/bun`)
- This makes TypeScript use Bun's native type definitions

### 5. Optimize templates/bun/bunfig.toml

- Add settings optimized for Ink CLI development:
  - `auto-install = true` for automatic deps
  - `run = { silent = false }` for interactive TUI
  - Keep build optimizations for production
  - Add Ink/React specific loader configs if needed

### 6. Delete unused templates

- Delete entire `templates/js/` directory
- Delete entire `templates/ts/` directory
- These are no longer needed since we only support Bun

### 7. Update test.js

- Remove JS and TS template tests
- Keep only Bun template test (or adapt to test Bun template)
- Update test to use `bun run` commands instead of `npm run`

---

## Files to Modify

- `cli.js`
- `index.js`

- `templates/bun/tsconfig.json`
- `templates/bun/bunfig.toml`
- `test.js`

## Files to Delete

---

## Expected Result

After changes:

- `create-ink-app-with-bun my-app` → Uses Bun template by default
- No flags required
- Generated project uses native Bun types instead of @types/node
- bunfig.toml optimized for Ink CLI development
